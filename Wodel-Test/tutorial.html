Wodel-Test

A Model-based Framework for Language-independent Mutation Testing

View on GitHub

Role of WodelTestGlobalResult, WodelTestResult, WodelTestResultClass, and WodelTestInfo classes:

WodelTestInfo(String testName, boolean value, String info, String message)

WodelTestInfo.java

testName: the name of the test (this is an atomic test case, it can be included in a file with more test cases)
value: whether the test has passed (mutant is live) : false, or the test has not passed (mutant is killed) : true
info: some other string info that you can need, like the test file path
message: the error message from the test in case it has not passed

WodelTestResult(String className, String modelPath, List<Object> tests, List<WodelTestInfo> info)

WodelTestResult.java

className: the name of the class containing the tests
modelPath: the path to the model that has been tested
tests: the tests that had been executed (although it is a List<Object> variable, it is used as a List<String>)
info: a list of the single test cases corresponding to this test file and their results in the format of the previous WodelTestInfo class

WodelTestResultClass

WodelTestResultClass.java

Collection of the mutation testing results so far. When each test file finishes its execution, you need to update it with:

WodelTestResultClass resultClass = WodelTestResultClass.getWodelTestResultClassByName(results, modelPath);
if (resultClass == null) {        //if we need a new WodelTestResultClass object
        resultClass = new WodelTestResultClass(modelPath);
        results.add(resultClass);      //we add the new WodelTestResultClass object to the WodelTestGlobalResult results list
}
resultClass.addResult(wtr); //wtr is the WodelTestResult object obtained in the previous step

WodelTestGlobalResult

WodelTestGlobalResult.java

This is the root class for the mutation testing process results. It is the object that the run() method returns. We only need to create it at the beginning of the run() method:

WodelTestGlobalResult globalResult = new WodelTestGlobalResult();
List results = globalResult.getResults();

In case the Wodel-Test views do not show properly, proceed as follows:

1. Close all of them
2. Select the SUT project in the Model Explorer
3. Reopen the Wodel-Test views with the SUT project selected in the Model Explorer
